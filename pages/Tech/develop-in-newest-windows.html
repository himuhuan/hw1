士别三日：现在在Windows开发还是如此不堪吗？

发布者：Himu

前言

众所周知，Windows因为包管理，第三方库安装复杂等问题被各路开发者（尤其是非C#系）所不屑， 并纷纷转投Linux，或者Mac OS等其他操作系统。

然而，今非昔比，现如今Microsoft的策略向开源靠拢。现在的Windows有着许多不被许多程序员所知的工具， 在经过适当的调教后，笔者个人认为整体的开发体验不逊于Unix系。

终端配置

在以前，Windows 最为人诟病的便是糟糕的命令行体验。现在，令人欣慰的是， 微软官方已在最新的Windows上内置了现代的Terminal程序。 搜索 Terminal 即可打开它，你可以在设置里将其设为默认Terminal程序，告别丑陋的cmd.exe!

Windows Terminal 默认状态，可以看到它支持多标签页，快捷命令等现代终端功能

说真的，这是一款令人惊艳的 Terminal 程序，尝试靠 官方文档 配置它。 依靠PowerShell本身强大的包管理，可以很轻松装上在Linux/Mac上熟悉的Powerline，并且将个性最大化！

本项目在Terminal内使用git中的预览，可以看到 Powerline 对 git 有着良好的支持

包管理器 --- 以 Scoop 为例

说起配置 Windows 的开发环境 --- 这有时真的是一场噩梦，但是现在，许多第三方都为 Windows 提供了包管理支持。 下表罗列了常见可用的包管理与其特色。

常见包管理器汇总
名称 提供商 与 Windows 生态对接 可扩展性
Scoop 开源社区 一般 好
Chocolatey 一般
Winget 微软 好 一般

读者可以根据需求选择, 笔者使用的是 scoop . 如图所示，再配置后安装程序是一件很简单的事情，scoop 也会默认配置环境变量

库管理 --- vcpkg

以前，Windows上处理 C/C++ 第三方库依赖也是棘手的问题，同样的，微软现在意识到了这一点， 并推出了 vcpkg 处理这个问题，而且 vcpkg 与 Visual Studio 有着友好的集成性，详情参见 vcpkg 官网
